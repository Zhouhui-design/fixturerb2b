# 自动连接管理员功能 - 快速测试指南

## 部署后立即测试

### 1. 清除缓存（必须！）

**电脑端:**
- Chrome/Firefox: `Ctrl + Shift + R`
- Safari: `Cmd + Shift + R`

**手机端:**
- iOS Safari: 长按刷新按钮 2-3 秒 → 选择"清空缓存并重新加载"
- Android Chrome: 点击菜单 → 刷新图标 → "清空缓存并硬性重新加载"

**或者使用隐身模式:**
- Chrome: `Ctrl + Shift + N` (Windows) / `Cmd + Shift + N` (Mac)
- Firefox: `Ctrl + Shift + P` (Windows) / `Cmd + Shift + P` (Mac)

## 常见问题排查

### 问题 1: 点击按钮没反应

**检查:**
1. 调试面板是否有日志输出？
2. 是否有红色错误信息？
3. Network 标签中 `/api/auth/register` 请求是否成功？

**可能原因:**
- JavaScript 文件未加载（检查 Console 是否有 404 错误）
- Service Worker 缓存了旧版本（强制刷新）
- 后端服务未启动（检查 PM2 状态）

---

### 问题 2: 登录后显示用户列表而不是聊天窗口

**检查:**
1. 调试面板中 `isAdmin` 的值是什么？
2. 是否看到"检测到普通用户，开始自动连接管理员..."？
3. `/api/auth/admin` API 是否返回了数据？

**可能原因:**
- 后端返回的 `isAdmin` 字段缺失或为 `true`
- User 模型中缺少 `isAdmin` 字段定义
- MongoDB 中 Admin 用户的 `isAdmin` 不是 `true`

**修复:**
```javascript
// 在 MongoDB 中检查
db.users.find({username: "Admin"})

// 如果不是 true，更新它
db.users.updateOne(
  {username: "Admin"},
  {$set: {isAdmin: true}}
)
```

---

### 问题 3: 仍然加载旧版本代码

**解决方案:**
1. **强制刷新**: Ctrl+Shift+R
2. **清除缓存**: 浏览器设置 → 隐私和安全 → 清除浏览数据
3. **硬刷新**: 打开 DevTools → 右键点击刷新按钮 → "清空缓存并硬性重新加载"
4. **隐身模式**: 使用隐身/隐私模式窗口

**验证是否加载新版本:**
- 打开页面后应该立即看到调试信息面板
- 如果没有看到，说明还在使用旧版本

---

### 问题 4: 手机端无法调试

**解决方案:**
现在页面上有调试信息面板，可以直接看到所有日志！

**如何查看:**
1. 登录页面底部会显示调试面板
2. 所有关键操作都会记录在面板中
3. 点击"清除日志"按钮可以清空历史记录
